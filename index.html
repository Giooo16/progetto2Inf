Argomenti Principali:

- Ereditarietà da Exception Le eccezioni personalizzate ereditano dalla classe base Exception, mantenendo compatibilità con il sistema di gestione errori di Python e permettendo l'uso di try/except standard.

- Errori Specifici del Dominio Ogni eccezione rappresenta uno stato di errore particolare dell'applicazione PulseNews, migliorando la leggibilità e la manutenibilità del codice attraverso messaggi di errore contestualizzati.

- Gestione Gerarchica Utilizzo di una gerarchia di eccezioni con PulseNewsError come base, permettendo catch selettivi e una gestione modulare degli errori in diversi livelli dell'applicazione.

- Messaggi Informativi Le eccezioni includono messaggi dettagliati per facilitare il debugging e la comprensione del problema, fornendo informazioni utili per la risoluzione.

class PulseNewsError(Exception):

Modello Articolo

Argomenti Principali:

- DTO Pattern La classe Articolo incapsula i dati di una notizia in un oggetto strutturato, facilitando il passaggio e la manipolazione dei dati nell'applicazione senza usare dizionari caotici, migliorando la type safety.

- Property e Validazione Utilizzo di @property per titolo con setter che valida l'input, prevenendo titoli vuoti o invalidi attraverso eccezioni e garantendo l'integrità dei dati.

- Uguaglianza Personalizzata Override di __eq__ per definire l'uguaglianza basata su titolo e fonte, essenziale per gestire duplicati nel Catalogo e ottimizzare le operazioni di set.

- Serializzazione JSON Metodo to_dict per convertire l'oggetto in dizionario, compatibile con JSON per salvataggio o trasmissione dati, semplificando l'interfaccia con fonti esterne.

- Gestione Temporale Conversione automatica della data da stringa ISO a oggetto datetime per operazioni di confronto, ordinamento e filtri temporali, utilizzando la libreria standard datetime.

- Attributi Opzionali Supporto per tags come lista dinamica e rilevanza come punteggio float, con valori di default per flessibilità e personalizzazione delle notizie.

Python
class Articolo:
    def __init__(self, id: str, titolo: str, corpo: str, autore: str,
                 fonte: str, data_pubblicazione: str, categoria: str,
                 tags: list = None, rilevanza: float = 0.5):
        self._id = id
        self.titolo = titolo  # Chiama il setter per la validazione
        self.corpo = corpo    # Chiama il setter per la validazione
        self._autore = autore
        self._fonte = fonte
        self._data_pubblicazione = datetime.fromisoformat(data_pubblicazione)
        self._categoria = categoria
        self.tags = tags if tags is not None else []
        self.rilevanza = rilevanza
        
    @property
    def titolo(self) -> str:
        return self._titolo

    @titolo.setter
    def titolo(self, valore: str):
        if not valore or not valore.strip():
            raise ValueError("Il titolo non può essere vuoto.")
        self._titolo = valore

    def __eq__(self, altro):
        """Due articoli sono uguali se hanno stesso titolo E stessa fonte"""
        if not isinstance(altro, Articolo):
            return False
        return self.titolo == altro.titolo and self.fonte == altro.fonte

    def to_dict(self) -> dict:
        """Serializza l'articolo in un dizionario JSON-friendly."""
        return {
            "id": self.id,
            "titolo": self.titolo,
            "data_pubblicazione": self.data_pubblicazione.isoformat(),
            "categoria": self.categoria
        }

Gestione Fonti

Argomenti Principali:

- Principio Aperto/Chiuso L'architettura permette l'estensione con nuove fonti senza modificare il codice esistente, grazie all'ereditarietà, seguendo i principi SOLID per un design scalabile.

- Classi Astratte (ABC) La classe Fonte impone l'implementazione del metodo fetch_articoli() nelle sottoclassi concrete, garantendo un'interfaccia uniforme.

- Polimorfismo Diverse fonti (JSON, API, ecc.) implementano fetch_articoli() in modi specifici, trattate uniformemente dal Catalogo attraverso riferimenti alla classe base.

- Gestione Stato Attributo 'attiva' per abilitare/disabilitare fonti dinamicamente senza rimuoverle, permettendo configurazioni flessibili dell'applicazione.

Python
class Fonte(ABC):
    def __init__(self, nome: str, url: str, categoria_default: str, attiva: bool = True):
        self._nome = nome
        self._url = url
        self._categoria_default = categoria_default
        self._attiva = attiva

    @abstractmethod
    def fetch_articoli(self) -> list:
        pass

class FonteJSON(Fonte):
    def __init__(self, nome, url, cat, percorso_file, attiva=True):
        super().__init__(nome, url, cat, attiva)
        self._percorso_file = percorso_file

    def fetch_articoli(self) -> list:
        try:
            with open(self._percorso_file, 'r', encoding='utf-8') as f:
                dati_grezzi = json.load(f)
                articoli_list = []
                for item in dati_grezzi.get("articoli", []):
                    articolo = Articolo.from_dict(item, self.nome)
                    articoli_list.append(articolo)
                return articoli_list
        except (FileNotFoundError, json.JSONDecodeError) as e:
            raise FonteNonRaggiungibileError(f"Errore caricamento: {e}")

Sistema di Filtri

Argomenti Principali:

- Programmazione Funzionale Uso di funzioni lambda e built-in come filter() per applicare criteri di filtraggio in modo dichiarativo, evitando cicli imperativi.

- Concatenazione di Filtri Possibilità di applicare più filtri in sequenza, concatenandoli elegantemente, per query complesse senza codice ripetitivo.

- Criteri Dinamici I filtri possono essere passati come argomenti, permettendo flessibilità nell'interrogazione e personalizzazione runtime.

- Metodi Statici Tutti i metodi sono statici per un uso stateless e modulare del sistema di filtri, facilitando il testing e il riutilizzo.

Python
class Filtri:
    """Collezione di metodi statici per filtrare liste di articoli."""

    @staticmethod
    def applica_filtri(articoli: list, filtri: list) -> list:
        for f in filtri:
            articoli = list(filter(f, articoli))
        return articoli

    @staticmethod
    def filtra_per_categoria(categoria: str):
        """Restituisce una lambda function che filtra per la categoria esatta."""
        return lambda art: art.categoria.lower() == categoria.lower()

    @staticmethod
    def filtra_recenti(giorni: int):
        """Restituisce articoli pubblicati negli ultimi X giorni."""
        oggi = datetime.now()
        return lambda art: (oggi - art.data_pubblicazione).days <= giorni

Analizzatore Testuale

Argomenti Principali:

- Espressioni Regolari (Regex) Utilizzo di re.findall per estrarre parole alfabetiche dal testo, rimuovendo punteggiatura e caratteri speciali per un'analisi pulita.

- Rimozione Stop-Words Filtro funzionale per escludere parole comuni inutili, migliorando la qualità delle keyword e riducendo il rumore nei risultati.

- Conteggio Frequenze Classe Counter per calcolare le occorrenze e restituire le top keyword, sfruttando strutture dati efficienti per prestazioni ottimali.

- Pulizia Testo Rimozione di URL e caratteri non alfanumerici per un'analisi più accurata, utilizzando regex per pattern complessi.

Python
class Analizzatore:
    STOP_WORDS = {"il", "la", "e", "di", "in", "per", "con", "su", "un", "una"}

    @staticmethod
    def estrai_keyword(testo: str, top_n: int = 5) -> list:
        # 1. Rimuove URL
        testo_pulito = re.sub(r"http\S+|www\S+", "", testo)
        
        # 2. Estrae solo le parole alfabetiche (no punteggiatura/numeri)
        parole = re.findall(r"\b[a-zA-Zàèéìòù]{4,}\b", testo_pulito.lower())
        
        # 3. Applica un filtro funzionale per escludere le stop words
        parole_utili = list(filter(lambda w: w not in Analizzatore.STOP_WORDS, parole))
        
        # 4. Usa Counter per le frequenze e restituisce le prime top_n
        conteggio = Counter(parole_utili)
        return [parola for parola, freq in conteggio.most_common(top_n)]

Motore Cercatore

Argomenti Principali:

- List Comprehensions Sintassi concisa per filtrare articoli che matchano la regex query, combinando filtri e trasformazioni in una singola espressione.

- Regex Search re.search per ricerche case-insensitive nel titolo e corpo degli articoli, supportando pattern complessi per query avanzate.

- Ordinamento Dinamico sorted() con key lambda per ordinare per data o altri criteri, permettendo ordinamenti personalizzati senza codice aggiuntivo.

- Storico Ricerche Tracciamento delle query eseguite per analisi e ottimizzazioni future, memorizzando statistiche per migliorare le prestazioni.

Python
class Cercatore:
    def __init__(self, catalogo):
        self._catalogo = catalogo
        self._storico_ricerche = []

    def cerca_testo(self, regex_query: str, ignora_case: bool = True) -> list:
        try:
            flags = re.IGNORECASE if ignora_case else 0
            pattern = re.compile(regex_query, flags)
            
            risultati = [
                art for art in self._catalogo.articoli
                if pattern.search(art.titolo) or pattern.search(art.corpo)
            ]
            
            self._storico_ricerche.append((regex_query, len(risultati)))
            return risultati
        except re.error:
            raise QueryNonValidaError(f"Regex non valida: {regex_query}")

    def ordina_per_data(self, articoli: list, decrescente: bool = True) -> list:
        """Ordina la lista degli articoli per data_pubblicazione"""
        return sorted(articoli, key=lambda a: a.data_pubblicazione, reverse=decrescente)

Catalogo Centrale

Argomenti Principali:

- Singleton Pattern Il Catalogo è l'unica fonte di verità in memoria per gli articoli, centralizzando l'accesso ai dati.

- Set per Unicità Utilizzo di set() per garantire assenza di duplicati, basandosi su __eq__, ottimizzando le operazioni di inserimento.

- Gestione Fonti Registro delle fonti per fetch centralizzato, permettendo l'aggiunta dinamica di nuove fonti dati.

- Property Sicura Esposizione degli articoli come lista immutabile per proteggere l'integrità dei dati e prevenire modifiche accidentali.

Python
class Catalogo:
    def __init__(self):
        self._articoli = set()  # Usiamo un Set per evitare duplicati in automatico
        self._fonti = {}

    @property
    def articoli(self) -> list:
        # Espone una vista lista sicura
        return list(self._articoli)

    def aggiungi_articoli(self, nuovi_articoli: list) -> int:
        """Aggiunge una lista di articoli e ritorna il numero di aggiunte effettive"""
        conteggio_iniziale = len(self._articoli)
        self._articoli.update(nuovi_articoli)
        return len(self._articoli) - conteggio_iniziale

    def registra_fonte(self, fonte: Fonte):
        self._fonti[fonte.nome] = fonte

Fetcher Asincrono

Argomenti Principali:

- Programmazione Asincrona asyncio per gestire I/O concorrenti senza bloccare il thread principale, migliorando la responsività dell'applicazione.

- Coroutine Funzioni async def per operazioni non bloccanti, permettendo l'esecuzione parallela di task I/O-bound.

- Gather per Parallelismo asyncio.gather per eseguire fetch da multiple fonti contemporaneamente, riducendo il tempo totale di caricamento.

- Gestione Errori Asincrona Cattura e gestione degli errori specifici durante il fetch concorrente, mantenendo la stabilità del sistema.

Python
class FetcherAsync:
    def __init__(self, catalogo):
        self.catalogo = catalogo

    async def _fetch_da_fonte(self, fonte: Fonte):
        """Esegue l'estrazione dati simulando un'attesa I/O asincrona"""
        if not fonte.attiva: return
        try:
            await asyncio.sleep(0.5) # Simulazione latenza di rete API
            articoli = fonte.fetch_articoli()
            aggiunti = self.catalogo.aggiungi_articoli(articoli)
            print(f"[ASYNC] Fonte '{fonte.nome}' ha restituito {aggiunti} articoli.")
        except FonteNonRaggiungibileError as e:
            print(f"[ASYNC-ERR] {e}")

    async def esegui_fetch_massivo(self):
        """Lancia contemporaneamente il fetch per tutte le fonti registrate."""
        tasks = []
        for fonte in self.catalogo._fonti.values():
            tasks.append(self._fetch_da_fonte(fonte))
            
        # gather attende la risoluzione in parallelo di tutte le Coroutine
        await asyncio.gather(*tasks)

Thread di Monitoraggio

Argomenti Principali:

- Multithreading threading per eseguire monitoraggio in background senza bloccare l'UI, permettendo operazioni simultanee.

- Daemon Thread Thread daemon che termina automaticamente con il programma, evitando risorse orfane.

- Sincronizzazione Lock per evitare race conditions, Event per shutdown graceful, garantendo thread safety.

- Loop di Monitoraggio Ciclo continuo con timeout per controlli periodici senza consumo eccessivo di CPU, bilanciando efficienza e reattività.

Python
class ThreadMonitoraggio(threading.Thread):
    def __init__(self, catalogo):
        super().__init__(daemon=True)
        self.catalogo = catalogo
        self._stop_event = threading.Event()

    def run(self):
        """Metodo invocato quando il thread fa 'start()'"""
        while not self._stop_event.is_set():

                break

            with self._lock:

                pass
                
    def ferma(self):
        """Segnala al thread di interrompersi pulitamente."""
        self._stop_event.set()

L'App PulseNews

Argomenti Principali:

- Controller Pattern PulseNews orchestra i componenti e gestisce l'interazione utente, fungendo da punto centrale dell'architettura.

- Event Loop Asincrono asyncio.run per il caricamento iniziale, integrando operazioni sincrone e asincrone.

- Menu Interattivo Loop sincrono per input utente mentre thread girano in background, fornendo un'interfaccia testuale semplice.

- Integrazione Componenti Collegamento tra Catalogo, Cercatore, Fetcher e Thread per un funzionamento coeso e modulare dell'applicazione.

Python
class PulseNews:
    def __init__(self):
        self._catalogo = Catalogo()
        self._cercatore = Cercatore(self._catalogo)
        self._thread_monitoraggio = None

    def avvia(self):
        print(" Inizializzazione applicazione...")
        # ... Popolamento Fonti omesse per brevità ...

        # 1. Avvio Event Loop Asincrono per il primo caricamento dati pesante
        fetcher = FetcherAsync(self._catalogo)
        asyncio.run(fetcher.esegui_fetch_massivo())

        # 2. Avvio worker thread in background
        self._thread_monitoraggio = ThreadMonitoraggio(self._catalogo)
        self._thread_monitoraggio.start()

        # 3. Main Loop Sincrono per interazione utente
        self.menu_interattivo()
        
    def menu_interattivo(self):
        while True:
            scelta = input("\nScegli un'opzione (es. 0 per uscire) > ")
            if scelta == "0":
                self._thread_monitoraggio.ferma()
                self._thread_monitoraggio.join()
                print("Chiusura. Arrivederci!")
                break
