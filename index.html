Eccezioni Personalizzate

Descrizione Generale: Questa sezione implementa il sistema di gestione degli errori personalizzati dell'applicazione PulseNews, definendo una gerarchia di eccezioni specifiche per gestire situazioni di errore particolari come query non valide o fonti non raggiungibili.

Argomenti Principali:

- Ereditarietà da Exception Le eccezioni personalizzate ereditano dalla classe base Exception di Python, garantendo piena compatibilità con il sistema di gestione degli errori standard del linguaggio. Questo permette di utilizzare i blocchi try/except tradizionali senza modifiche al codice esistente, mantenendo la semantica attesa dagli sviluppatori.

- Errori Specifici del Dominio Ogni eccezione rappresenta un tipo di errore specifico legato alla logica dell'applicazione PulseNews, come problemi di caricamento fonti o query non valide. Questo approccio migliora significativamente la leggibilità del codice, poiché i nomi delle eccezioni comunicano chiaramente il contesto dell'errore, facilitando la manutenzione e il debugging.

- Gestione Gerarchica degli Errori Viene implementata una gerarchia di eccezioni con PulseNewsError come classe base astratta. Questa struttura permette di catturare errori in modo selettivo: ad esempio, è possibile catturare tutti gli errori di PulseNews con un singolo except PulseNewsError, oppure gestire sottotipi specifici come QueryNonValidaError separatamente, abilitando una gestione modulare e granulare degli errori attraverso i diversi livelli dell'applicazione.

- Messaggi Informativi e Contestualizzati Le eccezioni includono messaggi di errore dettagliati e informativi che forniscono contesto specifico sull'errore avvenuto. Questi messaggi aiutano gli sviluppatori durante il debugging, indicando non solo che si è verificato un errore, ma anche quale operazione stava fallendo e suggerendo possibili cause, accelerando la risoluzione dei problemi.

Modello Articolo

Descrizione Generale: Questa sezione definisce il modello di dati Articolo, che rappresenta una singola notizia nell'applicazione PulseNews, incapsulando tutte le informazioni rilevanti come titolo, contenuto, autore, fonte e data di pubblicazione.

Argomenti Principali:

- Pattern DTO (Data Transfer Object) La classe Articolo funge da contenitore strutturato per i dati di una notizia, incapsulando tutte le proprietà rilevanti (titolo, corpo, autore, ecc.) in un singolo oggetto. Questo approccio evita l'uso di dizionari caotici o tuple, migliorando la type safety e rendendo il codice più espressivo e meno soggetto a errori.

- Proprietà con Validazione Utilizzo del decoratore @property per il titolo con un setter personalizzato che valida l'input in ingresso. Se il titolo è vuoto o composto solo da spazi, viene sollevata una ValueError, prevenendo la creazione di articoli invalidi e garantendo l'integrità dei dati fin dalla loro creazione.

- Uguaglianza Personalizzata per Duplicati Override del metodo __eq__ per definire l'uguaglianza tra articoli basata esclusivamente su titolo e fonte. Questo è essenziale per gestire duplicati nel Catalogo: due articoli con stesso titolo dalla stessa fonte sono considerati identici, permettendo l'uso efficiente di strutture dati come set() per evitare ridondanze.

- Serializzazione JSON Compatibile Il metodo to_dict() converte l'oggetto Articolo in un dizionario Python standard, compatibile con la serializzazione JSON. Questo facilita l'interfaccia con fonti esterne di dati, il salvataggio su file o la trasmissione via rete, semplificando l'integrazione con API esterne.

- Gestione Temporale Intelligente La data di pubblicazione viene automaticamente convertita da stringa ISO 8601 a oggetto datetime durante l'inizializzazione, sfruttando il modulo datetime della libreria standard. Questo permette operazioni di confronto cronologico, ordinamento temporale e filtri basati su date senza conversioni manuali.

- Attributi Opzionali e Flessibili Supporto per attributi dinamici come tags (lista di stringhe) e rilevanza (punteggio float), con valori di default appropriati. Questo permette personalizzazioni avanzate delle notizie, come categorizzazione tramite tag o ranking basato su rilevanza, mantenendo la flessibilità necessaria per diverse fonti di dati.

from datetime import datetime

class Articolo:
    def __init__(self, id: str, titolo: str, corpo: str, autore: str,
                 fonte: str, data_pubblicazione: str, categoria: str,
                 tags: list = None, rilevanza: float = 0.5):
        self._id = id
        self.titolo = titolo  # Chiama il setter per la validazione
        self._corpo = corpo
        self._autore = autore
        self._fonte = fonte
        self._data_pubblicazione = datetime.fromisoformat(data_pubblicazione)
        self._categoria = categoria
        self.tags = tags if tags is not None else []
        self.rilevanza = rilevanza
        
    @property
    def titolo(self) -> str:
        return self._titolo

    @titolo.setter
    def titolo(self, valore: str):
        if not valore or not valore.strip():
            raise ValueError("Il titolo non può essere vuoto.")
        self._titolo = valore

    @property
    def corpo(self) -> str:
        return self._corpo

    @property
    def autore(self) -> str:
        return self._autore

    @property
    def fonte(self) -> str:
        return self._fonte

    @property
    def data_pubblicazione(self) -> datetime:
        return self._data_pubblicazione

    @property
    def categoria(self) -> str:
        return self._categoria

    def __eq__(self, altro):
        """Due articoli sono uguali se hanno stesso titolo E stessa fonte"""
        if not isinstance(altro, Articolo):
            return False
        return self.titolo == altro.titolo and self.fonte == altro.fonte

    def to_dict(self) -> dict:
        """Serializza l'articolo in un dizionario JSON-friendly."""
        return {
            "id": self._id,
            "titolo": self.titolo,
            "corpo": self.corpo,
            "autore": self.autore,
            "fonte": self.fonte,
            "data_pubblicazione": self.data_pubblicazione.isoformat(),
            "categoria": self.categoria,
            "tags": self.tags,
            "rilevanza": self.rilevanza
        }

    @classmethod
    def from_dict(cls, data: dict) -> 'Articolo':
        """Crea un Articolo da un dizionario."""
        return cls(
            id=data["id"],
            titolo=data["titolo"],
            corpo=data.get("corpo", ""),
            autore=data.get("autore", "Sconosciuto"),
            fonte=data.get("fonte", "Fonte sconosciuta"),
            data_pubblicazione=data["data_pubblicazione"],
            categoria=data["categoria"],
            tags=data.get("tags", []),
            rilevanza=data.get("rilevanza", 0.5)
        )

Gestione Fonti

Descrizione Generale: Questa sezione implementa il sistema di gestione delle fonti di dati, permettendo all'applicazione PulseNews di caricare articoli da diverse origini come file JSON, API REST o database, attraverso un'architettura estensibile e uniforme.

Argomenti Principali:

- Principio Aperto/Chiuso (SOLID) L'architettura del sistema di fonti segue il principio Open/Closed: è possibile estendere il sistema aggiungendo nuove fonti (come API REST o database) senza modificare il codice esistente. Questo viene realizzato attraverso l'ereditarietà, dove nuove classi Fonte concrete ereditano da Fonte astratta, garantendo un design scalabile e modulare.

- Classi Astratte e Interfacce Uniformi La classe Fonte è definita come classe astratta utilizzando il modulo abc (Abstract Base Classes). Questo impone alle sottoclassi di implementare il metodo fetch_articoli(), garantendo un'interfaccia uniforme. Ogni fonte, indipendentemente dal tipo (JSON, API, database), espone lo stesso contratto pubblico.

- Polimorfismo e Trattamento Uniforme Diverse implementazioni di fonti (FonteJSON, FonteAPI, ecc.) possono essere trattate in modo polimorfico attraverso riferimenti alla classe base Fonte. Il Catalogo può iterare su una collezione di fonti eterogenee e chiamare fetch_articoli() senza conoscere i dettagli implementativi, semplificando il codice client.

- Gestione Dinamica dello Stato Ogni fonte ha un attributo 'attiva' booleano che permette di abilitare o disabilitare dinamicamente la fonte senza rimuoverla dalla configurazione. Questo è utile per manutenzione (disabilitare fonti temporaneamente non funzionanti) o personalizzazione (consentendo all'utente di scegliere quali fonti utilizzare).

Python
from abc import ABC, abstractmethod
import json

class Fonte(ABC):
    def __init__(self, nome: str, url: str, categoria_default: str, attiva: bool = True):
        self._nome = nome
        self._url = url
        self._categoria_default = categoria_default
        self._attiva = attiva

    @property
    def nome(self) -> str:
        return self._nome

    @property
    def url(self) -> str:
        return self._url

    @property
    def categoria_default(self) -> str:
        return self._categoria_default

    @property
    def attiva(self) -> bool:
        return self._attiva

    @abstractmethod
    def fetch_articoli(self) -> list:
        pass

class FonteJSON(Fonte):
    def __init__(self, nome, url, categoria_default, percorso_file, attiva=True):
        super().__init__(nome, url, categoria_default, attiva)
        self._percorso_file = percorso_file

    def fetch_articoli(self) -> list:
        try:
            with open(self._percorso_file, 'r', encoding='utf-8') as f:
                dati_grezzi = json.load(f)
                articoli_list = []
                for item in dati_grezzi.get("articoli", []):
                    articolo = Articolo.from_dict(item)
                    articoli_list.append(articolo)
                return articoli_list
        except (FileNotFoundError, json.JSONDecodeError) as e:
            raise FonteNonRaggiungibileError(f"Errore caricamento: {e}")

Sistema di Filtri

Descrizione Generale: Questa sezione implementa il sistema di filtri per articoli, permettendo di selezionare e restringere i risultati del catalogo in base a criteri specifici come categoria, data di pubblicazione o altri attributi, utilizzando approcci funzionali e concatenabili.

Argomenti Principali:

- Programmazione Funzionale Il sistema di filtri sfrutta paradigmi funzionali di Python, utilizzando funzioni lambda anonime e built-in come filter() per applicare criteri di filtraggio in modo dichiarativo. Invece di cicli imperativi che modificano liste in-place, si utilizzano espressioni che descrivono "cosa" filtrare piuttosto che "come" farlo, rendendo il codice più conciso e meno soggetto a errori.

- Concatenazione e Composizione di Filtri È possibile applicare più filtri in sequenza attraverso il metodo applica_filtri(), che prende una lista di funzioni filtro e le applica una dopo l'altra. Questo permette di comporre filtri complessi: ad esempio, filtrare prima per categoria, poi per data, poi per autore, creando pipeline di trasformazione dati flessibili e riutilizzabili.

- Criteri Dinamici e Runtime I filtri possono essere passati come argomenti a funzioni, permettendo di definire la logica di filtraggio dinamicamente durante l'esecuzione. Questo è particolarmente utile per interfacce utente dove l'utente può scegliere criteri arbitrari, o per sistemi di configurazione dove i filtri vengono letti da file di configurazione.

- Metodi Statici e Design Stateless Tutti i metodi della classe Filtri sono statici, eliminando la necessità di istanziare oggetti e rendendo il sistema stateless. Questo facilita il testing (non c'è stato da gestire), il riutilizzo (chiamate dirette senza setup) e la composizione (filtri possono essere passati come funzioni pure senza dipendenze nascoste).

Python
from datetime import datetime

class Filtri:
    """Collezione di metodi statici per filtrare liste di articoli."""

    @staticmethod
    def applica_filtri(articoli: list, filtri: list) -> list:
        for f in filtri:
            articoli = list(filter(f, articoli))
        return articoli

    @staticmethod
    def filtra_per_categoria(categoria: str):
        """Restituisce una lambda function che filtra per la categoria esatta."""
        return lambda art: art.categoria.lower() == categoria.lower()

    @staticmethod
    def filtra_recenti(giorni: int):
        """Restituisce articoli pubblicati negli ultimi X giorni."""
        oggi = datetime.now()
        return lambda art: (oggi - art.data_pubblicazione).days <= giorni

Analizzatore Testuale

Descrizione Generale: Questa sezione implementa l'analizzatore testuale che estrae e analizza le parole chiave più significative dal contenuto degli articoli, utilizzando tecniche di elaborazione del linguaggio naturale per identificare temi e argomenti principali.

Argomenti Principali:

- Espressioni Regolari Avanzate Utilizzo del modulo re di Python per estrarre parole alfabetiche dal testo attraverso re.findall() con pattern complessi. Il pattern r"\b[a-zA-Zàèéìòù]{4,}\b" identifica parole di almeno 4 caratteri, escludendo numeri, punteggiatura e caratteri speciali, fornendo una base pulita per l'analisi testuale.

- Rimozione Stop-Words Intelligente Implementazione di un filtro funzionale per escludere parole comuni (stop words) come articoli, preposizioni e congiunzioni che non aggiungono significato semantico. Questo migliora drasticamente la qualità delle keyword estratte, concentrandosi sui termini più rilevanti e riducendo il rumore nelle analisi statistiche.

- Conteggio Frequenze con Counter Utilizzo della classe Counter dalla libreria collections per calcolare automaticamente le occorrenze delle parole. Questa struttura dati efficiente fornisce il metodo most_common() per estrarre le top N parole più frequenti, ottimizzando le prestazioni per analisi su grandi volumi di testo senza implementazioni manuali.

- Pulizia Testo Completa Applicazione di regex per rimuovere URL (r"http\S+|www\S+") e caratteri non alfanumerici prima dell'analisi. Questo garantisce che l'estrazione delle keyword sia basata solo sul contenuto testuale significativo, escludendo elementi tecnici come link o markup che potrebbero distorcere i risultati.

Python
import re
from collections import Counter

class Analizzatore:
    STOP_WORDS = {"il", "la", "e", "di", "in", "per", "con", "su", "un", "una"}

    @staticmethod
    def estrai_keyword(testo: str, top_n: int = 5) -> list:
        # 1. Rimuove URL
        testo_pulito = re.sub(r"http\S+|www\S+", "", testo)
        
        # 2. Estrae solo le parole alfabetiche (no punteggiatura/numeri)
        parole = re.findall(r"\b[a-zA-Zàèéìòù]{4,}\b", testo_pulito.lower())
        
        # 3. Applica un filtro funzionale per escludere le stop words
        parole_utili = list(filter(lambda w: w not in Analizzatore.STOP_WORDS, parole))
        
        # 4. Usa Counter per le frequenze e restituisce le prime top_n
        conteggio = Counter(parole_utili)
        return [parola for parola, freq in conteggio.most_common(top_n)]

Motore Cercatore

Descrizione Generale: Questa sezione implementa il motore di ricerca che permette agli utenti di trovare articoli specifici nel catalogo utilizzando query basate su espressioni regolari, con supporto per ordinamento e tracciamento delle ricerche effettuate.

Argomenti Principali:

- List Comprehensions per Filtraggio Efficiente Sintassi concisa delle list comprehensions per filtrare articoli che corrispondono alla query regex in una singola espressione. Questa costrutto combina generazione di lista, condizione di filtro e trasformazione in modo elegante, sostituendo cicli for multi-riga con espressioni più leggibili e performanti.

- Ricerca Regex Case-Insensitive Utilizzo di re.search() con flag re.IGNORECASE per ricerche testuali flessibili che non distinguono maiuscole/minuscole. Questo permette query più user-friendly, dove "Tecnologia" trova anche "tecnologia" o "TECNOLOGIA", migliorando l'usabilità senza sacrificare precisione.

- Ordinamento Dinamico per Criteri Multipli Funzione sorted() con parametro key basato su lambda functions per ordinare risultati per data, rilevanza o altri attributi. Questo approccio permette ordinamenti runtime personalizzati: decrescente per data (più recenti prima), crescente per rilevanza, o alfabetico per titolo, senza codice condizionale complesso.

- Tracciamento Storico delle Ricerche Memorizzazione automatica delle query eseguite con relativi conteggi di risultati in una lista interna. Questo storico permette analisi di utilizzo (query più popolari), ottimizzazioni delle prestazioni (cache per query frequenti) e debugging (tracciamento comportamenti utente).

Python
import re

class Cercatore:
    def __init__(self, catalogo):
        self._catalogo = catalogo
        self._storico_ricerche = []

    def cerca_testo(self, regex_query: str, ignora_case: bool = True) -> list:
        try:
            flags = re.IGNORECASE if ignora_case else 0
            pattern = re.compile(regex_query, flags)
            
            risultati = [
                art for art in self._catalogo.articoli
                if pattern.search(art.titolo) or pattern.search(art.corpo)
            ]
            
            self._storico_ricerche.append((regex_query, len(risultati)))
            return risultati
        except re.error:
            raise QueryNonValidaError(f"Regex non valida: {regex_query}")

    def ordina_per_data(self, articoli: list, decrescente: bool = True) -> list:
        """Ordina la lista degli articoli per data_pubblicazione"""
        return sorted(articoli, key=lambda a: a.data_pubblicazione, reverse=decrescente)

Catalogo Centrale

Descrizione Generale: Questa sezione implementa il catalogo centrale che funge da repository principale per tutti gli articoli dell'applicazione, gestendo l'archiviazione, la prevenzione dei duplicati e l'accesso unificato ai dati provenienti da diverse fonti.

Argomenti Principali:

- Pattern Singleton per Unicità Il Catalogo implementa implicitamente il pattern Singleton essendo l'unica fonte di verità per gli articoli in memoria. Tutti i componenti dell'applicazione (Cercatore, Fetcher, Filtri) accedono alla stessa istanza, garantendo consistenza dei dati e evitando duplicazioni o stati inconsistenti tra moduli diversi.

- Set per Prevenzione Automatica Duplicati Utilizzo di un set() interno per memorizzare gli articoli, sfruttando l'uguaglianza personalizzata (__eq__) della classe Articolo basata su titolo+fonte. - Registro Fonti Centralizzato Mantenimento di un dizionario di fonti registrate per coordinare il fetch centralizzato. Ogni fonte è associata a un nome univoco, permettendo riferimenti diretti e gestione dinamica (aggiunta/rimozione fonti runtime) senza impatto sui componenti che le utilizzano.

- Property di Sicurezza per Incapsulamento Esposizione degli articoli attraverso una property che restituisce una lista immutabile (copia della lista interna). Questo protegge l'integrità del catalogo impedendo modifiche accidentali dall'esterno, mentre permette operazioni di lettura sicure e performanti.

Python
from datetime import datetime

class Articolo:
    def __init__(self):
        self._articoli = set()  # Usiamo un Set per evitare duplicati in automatico
        self._fonti = {}

    @property
    def articoli(self) -> list:
        # Espone una vista lista sicura
        return list(self._articoli)

    def aggiungi_articoli(self, nuovi_articoli: list) -> int:
        """Aggiunge una lista di articoli e ritorna il numero di aggiunte effettive"""
        conteggio_iniziale = len(self._articoli)
        self._articoli.update(nuovi_articoli)
        return len(self._articoli) - conteggio_iniziale

    def registra_fonte(self, fonte: Fonte):
        self._fonti[fonte.nome] = fonte

Fetcher Asincrono

Descrizione Generale: Questa sezione implementa il fetcher asincrono che gestisce il caricamento parallelo degli articoli da multiple fonti di dati contemporaneamente, sfruttando la programmazione asincrona per migliorare le prestazioni e la responsività dell'applicazione.

Argomenti Principali:

- Programmazione Asincrona con asyncio Utilizzo del modulo asyncio per gestire operazioni I/O concorrenti senza bloccare il thread principale. Le funzioni async def permettono di scrivere codice sequenziale che viene eseguito in modo non bloccante, migliorando la responsività dell'applicazione durante operazioni di rete intensive.

- Coroutine per Operazioni Non Bloccanti Implementazione di coroutine (funzioni async) per ogni fetch da fonte individuale. Ogni coroutine può sospendere l'esecuzione durante attese I/O (come letture da file o chiamate API), permettendo al loop di eventi di eseguire altre task contemporaneamente.

- Parallelismo con asyncio.gather() Utilizzo di asyncio.gather() per lanciare contemporaneamente il fetch da multiple fonti. Invece di processare le fonti sequenzialmente (tempo totale = somma dei tempi), vengono eseguite in parallelo (tempo totale ≈ tempo della fonte più lenta), drasticamente riducendo i tempi di caricamento.

- Gestione Errori in Contesto Asincrono Cattura e gestione selettiva degli errori durante il fetch concorrente, mantenendo la stabilità del sistema. Errori in una fonte non bloccano le altre, e vengono loggati appropriatamente senza interrompere l'intero processo di caricamento dati.

Python
import asyncio

class FetcherAsync:
    def __init__(self, catalogo):
        self.catalogo = catalogo

    async def _fetch_da_fonte(self, fonte: Fonte):
        """Esegue l'estrazione dati simulando un'attesa I/O asincrona"""
        if not fonte.attiva: return
        try:
            await asyncio.sleep(0.5) # Simulazione latenza di rete API
            articoli = fonte.fetch_articoli()
            aggiunti = self.catalogo.aggiungi_articoli(articoli)
            print(f"[ASYNC] Fonte '{fonte.nome}' ha restituito {aggiunti} articoli.")
        except FonteNonRaggiungibileError as e:
            print(f"[ASYNC-ERR] {e}")

    async def esegui_fetch_massivo(self):
        """Lancia contemporaneamente il fetch per tutte le fonti registrate."""
        tasks = []
        for fonte in self.catalogo._fonti.values():
            tasks.append(self._fetch_da_fonte(fonte))
            
        # gather attende la risoluzione in parallelo di tutte le Coroutine
        await asyncio.gather(*tasks)

Thread di Monitoraggio

Descrizione Generale: Questa sezione implementa il thread di monitoraggio che esegue controlli periodici in background per rilevare nuovi articoli o aggiornamenti dalle fonti, senza interferire con l'interazione utente principale.

Argomenti Principali:

- Multithreading per Background Processing Utilizzo del modulo threading per eseguire il monitoraggio in un thread separato, permettendo operazioni continue in background senza bloccare l'interfaccia utente principale. Questo è essenziale per applicazioni interattive dove l'utente deve poter continuare a utilizzare il sistema mentre processi automatici sono attivi.

- Thread Daemon per Pulizia Automatica Il thread di monitoraggio è configurato come daemon (daemon=True), garantendo che termini automaticamente quando il programma principale finisce. Questo evita thread orfani che potrebbero impedire la chiusura pulita dell'applicazione o consumare risorse dopo l'uscita.

- Sincronizzazione con Lock e Event Implementazione di threading.Lock per proteggere sezioni critiche da race conditions, e threading.Event per coordinare l'avvio/arresto del thread. Questo garantisce thread safety: il lock previene accessi concorrenti ai dati condivisi, mentre l'Event permette shutdown graceful senza busy waiting.

- Loop di Monitoraggio Efficiente Ciclo while continuo con Event.wait(timeout) per controlli periodici senza consumo eccessivo di CPU. Invece di cicli busy che controllano costantemente, il thread dorme fino al timeout o alla segnalazione di stop, bilanciando reattività (controlli frequenti) con efficienza (basso utilizzo risorse).

Python
import threading

class ThreadMonitoraggio(threading.Thread):
    def __init__(self, catalogo):
        super().__init__(daemon=True)
        self.catalogo = catalogo
        self._stop_event = threading.Event()
        self._lock = threading.Lock()

    def run(self):
        """Metodo invocato quando il thread fa 'start()'"""
        while not self._stop_event.is_set():
            # Sleep interrotta preventivamente se scatta l'evento
            self._stop_event.wait(timeout=5.0) 
            if self._stop_event.is_set():
                break

            # Simulazione rilevamento nuovo articolo (esegui in Safe-Zone)
            with self._lock:

Argomenti Principali:

- Pattern Controller per Orchestrazione La classe PulseNews funge da controller principale, orchestrando l'interazione tra tutti i componenti (Catalogo, Cercatore, Fetcher, Thread). Centralizza la logica di business e coordina le operazioni, fungendo da punto di ingresso unico per l'utente e garantendo un flusso di dati coerente attraverso l'applicazione.

- Event Loop Asincrono per Inizializzazione Utilizzo di asyncio.run() per eseguire il caricamento iniziale dei dati in modo asincrono. Questo permette di sfruttare il parallelismo del FetcherAsync durante l'avvio, riducendo i tempi di startup senza bloccare l'interfaccia, migliorando l'esperienza utente iniziale.

- Menu Interattivo Sincrono Implementazione di un loop sincrono per l'interazione utente basato su input() dalla console. Mentre il thread di monitoraggio gira in background, il main thread rimane disponibile per comandi utente, creando una separazione chiara tra operazioni automatiche (asincrone/thread) e interazioni utente (sincrone).

- Integrazione Coesa dei Componenti Collegamento strutturato tra Catalogo (storage), Cercatore (query), FetcherAsync (caricamento parallelo) e ThreadMonitoraggio (aggiornamenti automatici). Ogni componente ha responsabilità chiare ma collabora attraverso interfacce ben definite, creando un'architettura modulare e manutenibile.
